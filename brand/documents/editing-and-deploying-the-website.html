Editing & Deploying the Jeong AI Website

A practical walkthrough for making changes safely and shipping them to staging and production.

Prepared by Jeong AI · June 4, 2026 · Version 1.0

Contents

The Jeong AI website is a static HTML/CSS/JS site — no frameworks, no build step. You edit files directly and deploy by pushing to Git. This guide gets you from a clean checkout to a live change in production.

01The project at a glance

Everything is plain HTML, CSS, and JavaScript hosted on Azure Static Web Apps. There are no dependencies to install and nothing to compile — what you see in the file is what ships.

  • Pages live at the repo root (index.html, checkout and product pages, privacy.html, terms.html).
  • Styles are in css/, scripts in js/, and the brand kit in brand/.
  • Deployment is automatic through GitHub Actions — your branch decides where it lands.
Branch = environment develop deploys to staging; master deploys to production (jeongai.com); pull requests get their own preview URL.

02Set up & run it locally

  1. Clone the repository 
    git clone <repo-url>
cd ebyg-automation
  2. Open or serve the site Open index.html directly in a browser, or run a local server so links resolve cleanly: 
    python -m http.server 8080
# or:  npx serve .
  3. View it Visit http://localhost:8080. Edit any file and refresh — changes appear immediately.
Tip In VS Code, the Live Server extension auto-reloads the page every time you save. Right-click index.html → "Open with Live Server."

03Make a change and deploy it

  1. Start on develop Always work on develop — never commit directly to master unless explicitly told to. 
    git checkout develop
git branch --show-current   # confirm: develop
  2. Edit the files Make your changes directly in the HTML/CSS/JS. Preview locally before committing.
  3. Commit your work 
    git add .
git commit -m "Describe what changed"
  4. Push to staging Pushing develop triggers the staging deploy automatically. Review it on the Azure preview URL. 
    git push origin develop
  5. Promote to production Once staging looks right, merge develop into master. That deploys to jeongai.com. 
    git checkout master
git merge develop
git push origin master
Important A push to master goes live to customers. Verify on staging first, and never commit straight to master without review.
